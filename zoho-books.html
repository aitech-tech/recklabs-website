Setup Guide

1

Create a Zoho API App

Go to api-console.zoho.in → Add Client → Server-based Applications. Fill in:
Homepage URL: http://localhost
Authorized Redirect URI: http://localhost:8000/callback
Copy the Client ID and Client Secret.

Outside India? Use api-console.zoho.com instead.

2

Create a .env File

In the ai-agent/ folder, create a file named .env and paste:

ZOHO_CLIENT_ID=your_client_id_here
ZOHO_CLIENT_SECRET=your_client_secret_here
ZOHO_REDIRECT_URI=http://localhost:8000/callback
3

Run the Installer

Double-click installer\install.bat. It installs Python dependencies and registers the agent with Claude Desktop automatically.

4

Restart Claude Desktop

Fully quit Claude Desktop from the system tray (not just close the window), then reopen it. The agent's 66 tools will appear automatically.

5

Authenticate with Zoho

In Claude Desktop, type: Authenticate with Zoho Books. A browser window opens — log in and click Accept. Done. You only authenticate once; tokens refresh automatically.

How Authentication Works

  1. Agent starts a temporary local server on port 8000
  2. Zoho's authorization page opens in your browser
  3. You log in and approve the permissions
  4. Zoho sends an auth code to localhost:8000/callback
  5. Agent exchanges the code for access + refresh tokens
  6. Tokens are saved to storage/tokens.json — local only, never transmitted
  7. All future requests use saved tokens; refresh happens automatically on expiry
Headless flow: If the browser doesn't open automatically, ask Claude: "Give me the Zoho OAuth URL". Open it manually, approve, then paste the code parameter back to Claude.

Limitations

  • Write operations are permanent. Deleted records cannot be recovered through Claude. Use Zoho Books' own UI for sensitive deletions.
  • No bulk imports. Tools process one record at a time. For large data imports, use Zoho Books' CSV import feature directly.
  • API rate limits. Zoho Books API has daily and per-minute request limits. Heavy use may result in temporary throttling (429 errors).
  • No email sending. The agent can create and update records but cannot send invoices or documents by email on your behalf.
  • Single organization by default. If you have multiple Zoho Books organizations, the agent uses the first one. Pass organization_id explicitly to target a specific org.
  • No Zoho CRM in this build. This version is Zoho Books only. CRM support is coming soon.
  • Custom skills: simple format is list-only. Simple Word skills only generate list/read operations. For create/update/delete, use the advanced template or the direct tools.

Dos & Don'ts

✓ Do

  • Authenticate before running any data tools
  • Ask Claude to "list skills" to see what's available
  • Use "connection status" to verify before a session
  • Keep .env private — never share it or commit it to git
  • Use Word templates for custom skills — no JSON editing needed
  • Verify tax calculations with your accountant before filing
  • Check storage/agent.log if something isn't working

✗ Don't

  • Delete records through Claude unless absolutely certain — permanent
  • Share storage/tokens.json — it contains live API tokens
  • Use this agent for large data migrations or bulk imports
  • Assume tax calculations are accountant-verified
  • Commit .env or tokens.json to version control
  • Run create/update skills on production data without testing
  • Use the agent as a substitute for your accountant's advice
